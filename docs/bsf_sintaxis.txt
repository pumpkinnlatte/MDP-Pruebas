% 1. Declaración de un fluente de estado booleano:

%    - Para declarar un fluente de estado se utiliza el predicado state_fluent/1 y state_fluent/2
%    - args: 1. predicado (con argumentos o no) los cuales definan las propiedades del fluente de estado
             2. constante que permite definir explicitamente si se desea modelar como booleano o multivaluado (bsf o ads)

%   Nota: En la declaración de fluentes operan dos métodos distintos: por inferencia y por declaración explicita 

%    Ej. Definición por inferencia
    state_fluent(defectuoso). %Puede ser verdadero o falso, 
   
%    Ej. Definición explicita
    state_fluent(defectuoso, bsf). %Puede ser verdadero o falso, 

% 2. Declaración de un fluente de estado booleano mediante reglas intencionales

%   - Se permite el uso de reglas para facilitar la declaración de fluentes.
%     El objetivo de esto es poder definir fluentes por comprensión y no solo por extensión.
%     Se aprovecha el sistema de ´grounding´ de ProbLog.

%    Ej. Declaración mediante reglas por inferencia
    state_fluent(marketed(X)) :- person(X).

    person(thiago).
    person(denis).
    person(fabio).
    person(leliane).

%    La regla anterior deriva en los siguiente mediante el proceso de grounding/aterrizado:

    state_fluent(marketed(thiago)).
    state_fluent(marketed(denis)).
    state_fluent(marketed(fabio)).
    state_fluent(marketed(leliane)).

%    Al ser los valores de origen hechos probabilisticos independientes, el método de inferencia 
     debe de considerarlo como un fluente de estado booleano (BSF)

%    Ej. Declaración mediante reglas y constante explicita
    state_fluent(marketed(X), bsf) :- person(X).

    person(thiago).
    person(denis).
    person(fabio).
    person(leliane).

%    La regla anterior deriva en los siguiente mediante el proceso de grounding:

    state_fluent(marketed(thiago, bsf)).
    state_fluent(marketed(denis, bsf)).
    state_fluent(marketed(fabio, bsf)).
    state_fluent(marketed(leliane, bsf)).

% 3. Es importante dejar claro que el algoritmo de inferencia solo entra en juego cuando la aridad de state_fluent es 1.
%    Aunque tengan el mismo nombre de predicado, ProbLog distingue como terminos distintos si es que estos tienen una aridad diferente
%    state_fluent/1 != state_fluent/2
